RULES



1. Languages Allowed :  C and C++ .
2. The competition is exclusively for individuals (no groups or teams allowed).
3. There will be a 30 second cooldown timer after every solution submitted. While the timer is active, you cannot make any further submissions until the countdown goes to zero.

Contest Format
Ranking Criteria
Code Execution Verdict

Format:

There are 4 levels in the contest. For accessing a level, you need to have points greater than or equal to the level's threshold value.
Until you have the minimum number of required points, the corresponding levels will be locked.

A submission is judged as correct if your solution gives the correct answer to all testcases for that problem. No partial points are awarded.

Points will not be changed as the contest progresses.
Points required for Unlocking Levels :

Level 1: 0 points.  (Already Unlocked)
Level 2: 12 points.
Level 3: 30 points.
Level 4: 40 points.

Submitting a correct solution for a problem will award you the points associated with that problem.
Submissions which are not judged as correct will not affect your score.
Execution time of submissions greater than 2 seconds will lead to timeout.

Ranking:

The ranking is decided by the following (in order):

1. Current Level of participant.
2. Points earned on the current level.
3. Points earned on previous levels.
4. Time of most recent submission on current level.

Code Execution Verdict:

The NCC online judge gives the following different verdicts :

Timeout : Your code exceeded the time limit allocated for that problem. Think of a way to do the problem faster!

Wrong Answer : Your code's output did not match the right answer.

Compile Error : Your code did not compile successfully on our system.

Judgement denied : Your code contained malicious system calls that harm the content on our server. You are NOT supposed to try to break/harm the server in any way. If you repeat this, you will be banned.

Runtime Error : Your code compiled successfully but gave a runtime error. Examples of runtime errors: Memory limit exceeded, memory leak, divide by zero error(floating point exception), etc. Check your code again!

Correct : Your code produces the output that was expected!

In queue : Your code is yet to be executed by the online judge, it is in queue. This will change to one of the above verdicts after it has been compiled , executed and checked.

In all cases, the decision of the NCC Judge shall be final.
